# Code of Conduct

This project is for careful security analysis. Keep discussions evidence-based, specific, and safe to publish.

## Expected behavior

- Discuss claims with technical precision.
- Separate proven findings from risks and hypotheses.
- Avoid posting live exploit details for third-party systems.
- Do not post secrets, tokens, customer data, private logs, or unresolved vulnerability details.
- Treat disagreement as a chance to improve the heuristic through evidence and specific reasoning.

## Unacceptable behavior

- harassment or personal attacks,
- publishing private or sensitive data,
- encouraging unauthorized testing,
- turning methodology discussions into exploit drops,
- repeatedly ignoring maintainer safety guidance.

Maintainers may edit, hide, or remove unsafe content.
