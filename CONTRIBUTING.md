# Contributing

Contributions are welcome if they improve clarity, examples, false-positive handling, remediation guidance, or domain coverage.

## Good contributions

- clearer wording,
- synthetic examples,
- public case studies with links,
- false-positive criteria,
- remediation patterns,
- domain-specific checklists,
- terminology improvements,
- issue templates or release hygiene.

## Safety requirements

Do not submit:

- unresolved third-party vulnerability details,
- customer data,
- private bounty details,
- secrets, tokens, private logs, or internal hostnames,
- exploit instructions against systems you do not own or have permission to test.

If an example comes from a real issue, use only public, already-remediated information and cite the public source.

## Style

Prefer precise claims over dramatic claims.

Use:

```text
The return path appears under-validated.
```

Instead of:

```text
This is definitely critical.
```

Evidence first. Labels second. State the supporting facts before assigning severity or vulnerability labels.
